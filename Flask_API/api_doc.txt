# API Documentation

This document provides a summary of all the API endpoints available in the Flask project.

## Authentication API (`/`)

### `POST /login`
- **Description**: Authenticates a user and returns a JWT token. It can also be used to verify an existing token.
- **Request Body (Login)**: 
  ```json
  {
    "username": "your_username",
    "password": "your_password"
  }
  ```
- **Request Body (Verify Token)**:
  ```json
  {
    "token": "your_jwt_token"
  }
  ```
- **Response**: A JWT access token and its expiration time.

### `POST /logout`
- **Description**: Clears the access token cookie to log the user out.
- **Response**: A confirmation message.

### `POST /register`
- **Description**: Registers a new user.
- **Request Body**:
  ```json
  {
    "username": "your_username",
    "password": "your_password"
  }
  ```
- **Response**: A confirmation message.

## Database API (`/db`)

### `GET /db/tables`
- **Description**: Returns a list of table names in the public schema of the database.
- **Response**: A JSON object containing a list of table names.

### `GET /db/query`
- **Description**: Executes a safe, read-only SQL query.
- **Query Parameters**: `city` (string), `limit` (integer, optional).
- **Response**: A JSON object containing the query results.

### `POST /db/query`
- **Description**: Executes a safe, read-only SQL query.
- **Request Body**:
  ```json
  {
    "sql": "SELECT * FROM your_table WHERE condition = %s",
    "params": ["value"]
  }
  ```
- **Response**: A JSON object containing the query results.

### `GET /db/city`
- **Description**: Queries data for a specific city.
- **Query Parameters**: `city` (string, required), `table` (string, required), `limit` (integer, optional).
- **Response**: A JSON object containing the query results.

## Log API (`/log`)

### `GET /log/init`
- **Description**: Creates the `logs` table in the database if it doesn't already exist.
- **Response**: A confirmation message.

### `POST /log`
- **Description**: Adds a new log entry to the database.
- **Request Body**:
  ```json
  {
    "level": "INFO",
    "message": "This is a log message."
  }
  ```
- **Response**: A confirmation message.

--cost_per_vin // Done
SELECT 
    dealer_code, 
    dealer_address,
    DATE_TRUNC('month', TO_DATE(repair_date, 'DD-MM-YYYY')) AS month,
    model_series,
    service_type,
    SUM(CAST(REPLACE(total_cost, ',', '') AS DECIMAL)) AS total_cost_sum,
    ROUND(SUM(CAST(REPLACE(total_cost, ',', '') AS DECIMAL)) / NULLIF(COUNT(DISTINCT fin), 0), 2) AS cost_per_vin
FROM warranty_claims
GROUP BY dealer_code, dealer_address, month, model_series, service_type
ORDER BY dealer_code, month;
